/tm:workflows:auto-implement-tasks
1. 先通过 taskmaster-ai 获取当前正在执行的任务,每次仅完成一个 taskmaster任务
2. 编码规范严格遵守 @CLAUDE.md 禁止使用 map 对象和 object 对象，必须使用明确的字段设计，且restful 的请求必须是 post 和 requestBody的形式，另外接口的实现必须严格遵守规范开发,禁止做出任何行为的兼容开发
3. 禁止做出如下行为：过度设计导致功能复杂，引入过度复杂的开发代码
4. 遇到问题需要调用对应的 mcp 进行思考分析和文档查询
5. 禁止 mock 和 TODO
6. 功能实现中，你需要保证完整度
7. 避免功能重复实现
8. 禁止做出兼容行为，我们必须保证和 mock js 是一致的，且可以适当多一些字段
9. 当上面 8条都没有问题后，就进行 git 提交并推送，
10. 上面完成后，就可以进行下一个任务，将 taskmaster-ai 调用，并将当前任务设置为 done
