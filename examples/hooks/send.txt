/tm:workflows:auto-implement-tasks
0. 先关闭正在运行的项目
1. 先通过 taskmaster-ai 获取当前正在执行的任务
2. 编码规范严格遵守 @CLAUDE.md 禁止使用 map 对象和 object 对象，必须使用明确的字段设计，且restful 的请求必须是 post 和 requestBody的形式，另外接口的实现必须严格遵守规范开发,禁止做出任何行为的兼容开发
3. 目前你正在根据  @mock.js 来确定对应入参和出参是否一致的，其中出参需要忽略统一的 ApiResult
4. 禁止做出如下行为：过度设计导致功能复杂，引入过度复杂的开发代码
5. 遇到问题需要调用对应的 mcp 进行思考分析和文档查询
6. 禁止使用mapstrct，必须使用 lombok ，禁止在entity 以及 dto 和 req 中引入 get set 方法，必须使用 lombok 完成
7. 禁止 mock 和 TODO
8. 功能实现中，你需要保证完整度
9. 目前使用 jdk17  所以基本都是jakarta的，请勿设置为老的版本
10. 避免功能重复实现
11. 继续执行的任务，任务需要保证优先以  @mock.js 的核心入参和出参一致的为前提
12. 强制必须保证  @mock.js 的核心入参和出参一致的为前提
13. 禁止做出兼容行为，我们必须保证和 mock js 是一致的，且可以适当多一些字段
14. 注意当前的底层都是完整的状态，禁止创建新的实体，如果出现找不到对应的底层就进行跳过，不得对 dao 进行调整
15. 必须保证程序可以编译，程序可以执行 mvn clean package
16. 必须保证程序可以启动，每次启动等待 15s 后，检测接口是否是健康的，如果遇到不健康，已经修复他，不要因为时间的问题，就导致了功能偷懒或者未实现的状态
17. 必须保证服务启动后，日志没有任何的报错，如果有应该先修复，再重新执行 15和 16补
18. 当上面 17条都没有问题后，就进行 git 提交并推送，
19. 上面完成后，就可以进行下一个任务，将 taskmaster-ai 调用，并将当前任务设置为 done
